# 사용 가이드·README 단순화 + PWA 설치 비주얼화 (설계)

- 날짜: 2026-07-14
- 대상: `docs/index.html`(사용 가이드 = GitHub Pages / 앱 `/guide.html`), `README.md`, `README.en.md`
- 배경: 가이드·README 내용이 너무 복잡함. 핵심 가치가 드러나지 않고, 알림의 필수 전제인 PWA 설치가 텍스트로만 안내됨.

## 목표(성공 기준)

1. README에 사용 가이드 링크(`https://yoojungyu.github.io/agent-hub/`)가 눈에 띄게 있다.
2. 가이드·README가 "한눈에" 읽히도록 단순화되고, 핵심 가치가 강조된다.
3. **인증서 설치**와 **PWA(홈 화면 추가) 설치**가 모두 비주얼 단계로 안내된다 — 둘 다 알림의 필수 전제임이 분명하다.
4. 상세·예외 설명은 삭제하지 않고 `<details>` 접기로 이동해 정보 손실이 없다.
5. 앱 빌드가 통과하고 `guide.html`이 정상 복사된다.
6. push 후 `v1.0.0` 태그로 릴리스가 게시된다.

## 핵심 가치 문구 (강조 대상)

> 자리를 비워도, 폰으로 **승인·질문 답변·명령**까지 — 모두 믿을 수 있는 **로컬 네트워크(LAN)** 안에서,
> 내가 **PC에서 승인한 기기**만 접근·제어.

3-포인트 밴드: ① 같은 네트워크(LAN) 안에서만 · ② 자리 비워도 폰에서 원격 제어 · ③ 내가 승인한 기기만.

## 가이드 재구성 (기존 다크 테마·폰 목업·인증서 애니메이션 재사용)

| STEP | 내용 | 비주얼 |
|---|---|---|
| 01 | 같은 네트워크 접속 | 기존 유지 |
| 02 | 인증서 등록 `알림 필수①` | 기존 애니메이션 재생기 유지 |
| 03 | 앱 설치(홈 화면 추가·PWA) `알림 필수②` **신규** | Android/iOS 토글 + 폰 목업(핵심 탭 강조) |
| 04 | 기기 인증받기(PC 승인) | 기존 유지 |
| 05 | 세션 지켜보기 | 텍스트 축약 |
| 06 | 폰에서 원격 제어(질문 답변·권한 승인·🔔 알림·터미널) | 기존 질문카드 목업 유지 |

- STEP 06의 항목·"제한 사항" 6개 중 **필수만 본문**, 나머지(Clawd 충돌, 10분 시간창 세부, 세션 이어받기 셸 종료, 응답대기 범위, 재설치 시 인증서 유지 등)는 `<details>` 접기로 이동.
