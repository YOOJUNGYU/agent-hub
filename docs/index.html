STEP 01

같은 네트워크로 접속Connect on the same network

PC에서 Agent Hub를 실행하면 콘솔 상단에 접속 주소가 나옵니다. 휴대폰을 PC와 같은 네트워크(같은 공유기·랜, 유선/무선 무관)에 두고 그 주소를 브라우저에 입력하세요. Run Agent Hub on your PC — the console shows an access address at the top. Put your phone on the same network as the PC (same router/LAN, wired or wireless) and open that address in its browser.

  1. PC 콘솔의 주소 확인 — 예: https://192.168.0.10:47600Find the address in the PC console — e.g. https://192.168.0.10:47600
  2. 휴대폰 브라우저 주소창에 입력Type it into your phone's browser
  3. 보안 경고가 뜨면 고급 → 계속(이동)If a security warning appears, tap Advanced → Proceed
⚠️ 보안 경고는 자체 서명 인증서라서 나옵니다. 다음 단계에서 인증서를 등록하면 경고 없이 접속되고 홈 화면 추가(앱 설치)도 됩니다. ⚠️ The warning appears because the certificate is self‑signed. Installing it (next step) removes the warning and enables Add to Home Screen.

STEP 04

기기 인증받기Get your device approved

아무 휴대폰이나 몰래 보지 못하도록, 처음 접속하면 승인이 필요합니다. 폰에서 기기 이름을 넣고 요청하면 PC에서 승인합니다. So not just any phone can watch, the first visit needs approval. Enter a device name on the phone, then approve it on the PC.

  1. 폰: 기기 이름 입력 → 인증 요청 보내기Phone: enter a name → Request access
  2. PC 콘솔 기기 관리에서 요청 확인 → 승인PC console Devices tab → Approve
  3. 폰이 자동으로 모니터 화면으로 전환The phone switches to the monitor automatically

STEP 05

세션 지켜보기Watch the sessions

PC에서 돌아가는 Claude 세션이 카드로 보입니다. active(작업 중) · idle(대기) · ended(종료). 카드를 탭하면 실시간 활동 피드로 들어갑니다. 세션이 작업 중이면 카드·상세에 펄스 점 + 회전 상태어(예: Channeling…)로 살아있음을 보여주고, 세션 경과 시간과 누적 토큰(예: 11m 38s · 31.8k tok)도 함께 표시합니다. Each Claude session shows as a card: active · idle · ended. Tap a card for its live activity feed. When a session is working, the card and detail show a pulsing dot + a rotating status word (e.g. Channeling…) so you can see it's alive, along with the session's elapsed time and cumulative tokens (e.g. 11m 38s · 31.8k tok).

  1. 카드 = 세션(제목·프로젝트·현재 작업·상태)Card = a session (title, project, current task, status)
  2. 탭 → 도구 호출·메시지가 시간순으로 흐르는 피드Tap → a feed of tool calls and messages in order
  3. 최신 메시지를 보고 있으면 자동으로 따라 내려갑니다If you're at the bottom, it auto‑scrolls with new messages
  4. 목록을 아래로 당기면 새로고침됩니다Pull the list down to refresh

STEP 06

폰에서 원격 제어Control from your phone

폰에서 Claude를 원격으로 다룹니다 — 질문에 답하고, 위험한 작업을 허용/거부하고, 알림을 받습니다. 더 깊이 조작하려면 세션을 폰에 ⌨ 터미널로 붙일 수도 있어요(선택). Control Claude from your phone — answer its questions, allow/deny risky actions, and get alerts. For deeper control you can also attach a session as a ⌨ terminal on the phone (optional).

  1. 🔔 알림 켜기 — 폰에서 한 번 켜면(PC 훅 자동 설치) 앱을 닫아도 세션이 당신을 기다릴 때 푸시가 옵니다. 알림 본문에 Claude의 마지막 메시지가 담겨 완료인지 질문인지 바로 압니다. 같은 내용은 반복해서 울리지 않고, 메시지가 바뀔 때만 새로 알립니다.🔔 Enable alerts — turn it on once (the PC hook installs itself). Even with the app closed, a push arrives when a session needs you, carrying Claude's last message so you know at a glance if it's done or asking. The same message won't buzz again — you're re‑alerted only when it changes.
  2. 💬 질문에 답하기 — Claude가 선택지 질문을 하면 질문·답변 카드가 떠, 옵션이나 직접 입력을 골라 답변 보내기. 실행 중인 그 세션에 바로 반영(새 터미널 불필요).💬 Answer questions — a Q&A card pops up; pick an option or type your own and Send. Applied to the running session, no terminal needed.
  3. 🔐 권한 승인 — Bash·파일 편집 등 위험 작업 요청 시 🔐 카드에서 허용/거부. 그 세션에 바로 적용됩니다.🔐 Approve permissions — for risky tools (Bash, file edits…) a 🔐 card lets you Allow/Deny, applied to that session.
  4. 🟡 응답 대기중 — 당신이 답해야 하는 세션은 목록에서 앰버색 카드로 표시됩니다. 탭하면 답변 화면이 다시 열립니다.🟡 Waiting for you — sessions needing your reply turn amber in the list; tap to reopen the answer sheet.
  5. ⌨ (선택) 터미널 열기 — 세션을 폰에 터미널로 붙여 프롬프트·/슬래시 명령까지. 하단 특수키(Esc·Tab·화살표·Enter)로 메뉴 이동.⌨ (Optional) Open terminal — attach a session as a terminal for prompts and /slash commands; bottom keys (Esc, Tab, arrows, Enter) navigate menus.
⏱️ 원격 답변엔 시간 창이 있어요Remote answers have a time window
Claude가 질문·권한을 기다리는 기본 약 10분(600초) 안에서만 폰으로 답할 수 있습니다. 앱이 꺼져 있었더라도 이 창 안에 앱을 켜면 답할 수 있어요. 창이 지나면 대기중 카드를 열 때 “세션 터미널로 직접 답하거나 PC에서 답하라”는 안내가 대신 표시됩니다. You can answer from the phone only within the ~10 minutes (600s) Claude waits — even if the app was closed, launching it within that window still lets you answer. After it passes, opening the waiting card shows guidance to answer via the session terminal or on the PC.
🔗 세션 터미널 (PC ↔ 폰 공유·이어받기)Session terminal (shared / taken over)
PC 콘솔의 세션 터미널 탭에서 같은 세션을 열면 폰과 하나의 터미널을 실시간 공유합니다. 세션을 처음 가져오면 중복 실행을 막기 위해 원래 그 세션을 띄웠던 PC 쉘 창이 종료됩니다. 이어받을 때 이전 대화 이력을 다시 그리므로 잠깐 이력 불러오는 중…이 보입니다. 모든 통신은 LAN 안에서만 오갑니다. (임의 셸이 필요하면 PC 설정의 웹 터미널 허용—기본 꺼짐—을 켜세요.) Open the same session from the PC console's Session terminal tab to share one live terminal with the phone. The first time you take a session over, the original PC shell window that launched it closes (to avoid a duplicate‑process conflict). On resume it replays history, so a brief Loading history… shows first. All traffic stays inside your LAN. (For an arbitrary shell, enable Allow web terminal in PC settings — off by default.)
⚠️ Clawd on Desk와 동시 실행 주의Running alongside Clawd on Desk
두 앱이 같은 질문 훅을 함께 가로채면 답변이 전달되지 않습니다. Agent Hub 실행 시 Clawd on Desk가 켜져 있으면 종료할지 묻는 창이 뜹니다 — [예]면 닫고 계속, [아니오]면 유지하되 모바일에서 답할 때 “Clawd on Desk가 실행 중이라 답변을 전달할 수 없습니다” 안내가 나옵니다. If both apps intercept the same question hook, answers won't be delivered. On startup Agent Hub asks whether to close Clawd on Desk — [Yes] closes it; [No] keeps it, but answering from mobile then shows “Clawd on Desk is running, can't deliver the answer.”

📶 알림엔 인터넷이 필요해요Alerts need internet
앱을 닫아도 오는 푸시는 폰·PC 모두 인터넷이 필요합니다(브라우저 푸시 서비스 경유). 질문 상세는 종단간 암호화되어 알림에 바로 표시됩니다. iOS는 홈 화면에 추가한 PWA + iOS 16.4 이상에서만 동작합니다. Push while the app is closed needs internet on both phone and PC (via the browser's push service). The question detail is end‑to‑end encrypted and shown right in the notification. On iOS it works only for a Home‑Screen PWA on iOS 16.4+.
🔄 재설치·업데이트해도 인증서·승인은 유지Reinstall/update keeps cert & approvals
인증서와 기기 승인은 LocalAppData에 보관되어 재설치·업데이트에도 유지됩니다(재승인 불필요). 완전 삭제 후 재설치하거나 인증서를 수동 삭제한 경우에만 다시 받으세요 — 앱에 표시된 주소를 휴대폰 브라우저로 엽니다. 이 받기는 같은 LAN 안에서만 동작하며 HTTP로 제공되므로 신뢰하는 로컬 네트워크에서만 쓰세요. The certificate and device approvals live in LocalAppData, so they survive reinstalls and updates (no re‑approval). Re‑download only after a full uninstall + reinstall or if you manually deleted the cert — open the address shown in the app in your phone browser. That download works only on the same LAN and over HTTP, so use it only on a trusted local network.
🧱 첫 연결이 안 되면 방화벽First connection fails? Firewall
휴대폰에서 첫 접속이 안 되면 PC의 Windows 방화벽에서 해당 포트의 수신(인바운드) 허용이 필요할 수 있습니다. If the first connection from a phone fails, you may need to allow inbound connections for that port in the PC's Windows Firewall.
🟡 ‘응답 대기중’은 언제 뜨나요When does “Waiting for you” show?
사용자가 실제로 답해야 할 때(질문·권한·입력 대기)만 카드가 앰버색으로 표시되고, 서브에이전트 대기 등 배경 작업엔 표시되지 않습니다. 진행 중 권한요청 표시는 앱을 새로고침하면 초기화될 수 있습니다. The card turns amber only when you must respond (a question, a permission, or input); it is not shown for background work like waiting on a subagent. A pending permission highlight may reset if you reload the app.
