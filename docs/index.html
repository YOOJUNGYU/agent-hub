STEP 01

휴대폰에서 접속하기Connect from your phone

PC에서 Agent Hub를 실행하면 콘솔 상단에 접속 주소가 나옵니다. 집·회사라면 휴대폰을 PC와 같은 네트워크(같은 공유기·랜)에 두고 그 주소를 브라우저에 입력하세요. 다른 네트워크(외부·LTE)에서 접속하려면 아래 ‘VPN으로 어디서나 접속’ 단계를 먼저 진행하면 됩니다. Run Agent Hub on your PC — the console shows an access address at the top. At home/office, put your phone on the same network (same router/LAN) as the PC and open that address in its browser. To connect from a different network (remote / mobile data), set up ‘Connect from anywhere with a VPN’ below first.

  1. PC 콘솔의 주소 확인 — 예: https://192.168.0.10:47600Find the address in the PC console — e.g. https://192.168.0.10:47600
  2. 휴대폰 브라우저 주소창에 입력Type it into your phone's browser
  3. 보안 경고가 뜨면 고급 → 계속(이동)If a security warning appears, tap Advanced → Proceed
⚠️ 보안 경고는 자체 서명 인증서라서 나옵니다. 다음 단계에서 인증서를 등록하면 경고 없이 접속되고 홈 화면 추가(앱 설치)도 됩니다. ⚠️ The warning appears because the certificate is self‑signed. Installing it (next step) removes the warning and enables Add to Home Screen.
🌍 집·회사 밖에서도 접속하려면 NetBird·Tailscale 같은 VPN을 쓰세요. 설정 방법과 모바일 접속·IP 안내는 바로 아래 ‘VPN으로 어디서나 접속’에 정리되어 있습니다. 🌍 To connect from outside your home/office, use a VPN like NetBird or Tailscale. Full setup, mobile access, and the IP details are in ‘Connect from anywhere with a VPN’ right below.

🌍 VPN으로 어디서나 접속CONNECT FROM ANYWHERE · VPN

집·회사 밖에서도, VPN으로 어디서나 접속Connect from anywhere with a VPN

Agent Hub는 클라우드 서버가 없어 기본적으로 같은 네트워크에서만 보입니다. 하지만 NetBird나 Tailscale 같은 오버레이 VPN을 PC와 휴대폰 양쪽에 깔아 로그인하면, 두 기기가 하나의 가상 네트워크로 묶여 어느 네트워크에 있든(집·회사·카페·LTE) 접속할 수 있습니다. 둘 다 개인용은 무료이고, 통신은 여전히 내 기기끼리 직접(WireGuard 암호화) 오갑니다. 둘 중 하나만 고르면 됩니다. Agent Hub has no cloud server, so by default it's only reachable on the same network. But if you install an overlay VPN like NetBird or Tailscale on both the PC and the phone and sign in, the two devices join one virtual network and you can connect from any network — home, office, cafe, or mobile data. Both are free for personal use, and traffic still flows device‑to‑device (WireGuard‑encrypted). Pick just one.

🐦NetBird추천recommended

무료 클라우드로 바로, 원하면 내 서버로도.Free cloud out of the box, or host it yourself.

  1. PC: netbird.io에서 Windows용을 설치하고 실행 → Connect 클릭 → 브라우저가 열리면 로그인 PC: install the Windows app from netbird.io, launch it, click Connect, and sign in when the browser opens
  2. 휴대폰: Play 스토어·App Store에서 NetBird 앱을 설치하고 같은 계정으로 로그인 (자체 호스팅이면 메뉴 ☰ → Change Server에서 관리서버 주소 입력) Phone: install the NetBird app and sign in with the same account (self‑hosted: set your management server under ☰ → Change Server)
  3. 두 기기가 같은 오버레이 네트워크에 연결되고 기기마다 고정 IP(100.x.x.x)를 받습니다. PC의 IP는 NetBird 대시보드나 앱에서 확인 Both devices connect to one overlay network and each gets a fixed IP (100.x.x.x). See the PC's IP in the NetBird dashboard or app
  4. 휴대폰 브라우저에서 https://<PC의 NetBird IP>:<포트> 접속 On the phone browser open https://<PC's NetBird IP>:<port>
🐾Tailscale가장 쉬움easiest

계정 로그인만으로 끝. 서버가 필요 없어요.Just sign in — no server to run.

  1. PC: tailscale.com/download에서 Windows용을 설치하고, Google·Microsoft·GitHub·Apple 계정 중 하나로 로그인 PC: install the Windows app from tailscale.com/download and sign in with a Google / Microsoft / GitHub / Apple account
  2. 휴대폰: Play 스토어·App Store에서 Tailscale 앱을 설치하고 같은 계정으로 로그인 Phone: install the Tailscale app from the Play Store / App Store and sign in with the same account
  3. 두 기기가 자동으로 같은 사설망(tailnet)에 묶입니다. PC의 Tailscale IP(100.x.x.x)는 트레이 아이콘 또는 tailscale ip로 확인 Both devices join one private network (tailnet). Find the PC's Tailscale IP (100.x.x.x) in the tray icon or via tailscale ip
  4. 휴대폰 브라우저에서 https://<PC의 Tailscale IP>:<포트> 접속. MagicDNS를 켜면 IP 대신 https://<PC이름>:<포트> 같은 이름으로도 접속 가능 On the phone browser open https://<PC's Tailscale IP>:<port>. With MagicDNS on you can also use a name like https://<pc-name>:<port>
💡 “네트워크를 옮기면 IP가 달라지잖아” — 맞습니다. 카페·LTE로 옮기면 일반 IP(192.168.x.x)는 매번 바뀝니다. 하지만 VPN IP(100.x.x.x)는 기기마다 고정이라, 어느 네트워크에 있든 그대로예요. 그래서 이 VPN 주소로 한 번 홈 화면에 앱 설치(또는 북마크)해 두면 집에서든 밖에서든 항상 같은 주소로 열립니다. (Tailscale은 IP 대신 MagicDNS 이름을 쓰면 더 편합니다.) 💡 “But the IP changes when I switch networks” — right, your ordinary IP (192.168.x.x) changes on every network. But the VPN IP (100.x.x.x) is fixed per device and stays the same wherever you are. So install this VPN address to the Home Screen (or bookmark it) once and it opens with the same address from home or away. (Tailscale's MagicDNS name is even handier than the IP.)
✅ Agent Hub 인증서는 이 VPN IP 대역(100.64.0.0/10)을 자동 포함해 발급되므로 경고 없이 접속됩니다. VPN을 나중에 깔았다면 Agent Hub를 한 번 재시작하면 VPN IP까지 넣어 인증서가 갱신돼요. 그다음 STEP 02 인증서 등록 · STEP 03 앱 설치는 이 VPN 주소로 똑같이 진행하면 됩니다. 포트는 PC 콘솔 설정에서 지정합니다. ✅ Agent Hub's certificate automatically covers this VPN IP range (100.64.0.0/10), so there's no warning. If you set the VPN up later, restart Agent Hub once to reissue the cert including the VPN IP. Then do STEP 02 (install cert) & STEP 03 (install app) exactly the same way, just using the VPN address. Set the port in the PC console's Settings.

STEP 05

STEP 06

폰에서 원격 제어Control from your phone

폰에서 Claude를 원격으로 다룹니다 — 질문에 답하고, 위험한 작업을 허용/거부하고, 알림을 받습니다. Control Claude from your phone — answer its questions, allow/deny risky actions, and get alerts.

  1. 🔔 알림 켜기 — 폰에서 한 번 켜면(PC 훅 자동 설치) 앱을 닫아도 세션이 당신을 기다릴 때 푸시가 옵니다. 알림 본문에 Claude의 마지막 메시지가 담겨 완료인지 질문인지 바로 압니다. 같은 내용은 반복해서 울리지 않고, 메시지가 바뀔 때만 새로 알립니다.🔔 Enable alerts — turn it on once (the PC hook installs itself). Even with the app closed, a push arrives when a session needs you, carrying Claude's last message so you know at a glance if it's done or asking. The same message won't buzz again — you're re‑alerted only when it changes.
  2. 💬 질문에 답하기 — Claude가 선택지 질문을 하면 질문·답변 카드가 떠, 옵션이나 직접 입력을 골라 답변 보내기. 실행 중인 그 세션에 바로 반영(새 터미널 불필요).💬 Answer questions — a Q&A card pops up; pick an option or type your own and Send. Applied to the running session, no terminal needed.
  3. ⌨️ 자유 답변 입력 — 세션 상세 화면 하단 입력창은 여러 줄로 쓸 수 있습니다(Enter=줄바꿈, 전송은 전송 버튼으로만). 전송하면 질문 카드 없이도 PC에서 실행 중인 그 세션 콘솔에 그대로 입력되고, 전송 중에는 입력창이 잠기고 로딩이 표시되며 전달이 확인돼야 다시 입력할 수 있어 중복 전송을 막습니다. 내가 보낸 답변은 오른쪽 말풍선으로 구분되어 표시됩니다.⌨️ Free‑text reply — the input bar at the bottom of the session detail screen supports multiple lines (Enter=newline; send only via the Send button). Hitting Send types it straight into that session's console on the PC, even without a pending question card; while sending, the input bar locks and shows a loading state, and only unlocks once delivery is confirmed — preventing duplicate sends. Replies you send show up in a right‑aligned bubble so they're easy to spot.
  4. 🔌 세션연결 — Windows Terminal에서 실행했거나 이미 종료된 세션처럼 직접 입력을 받을 수 없는 세션은 입력창 대신 ‘세션연결’ 버튼이 뜹니다. 누르면 PC에서 그 세션이 claude --resume으로 다시 열려, 이후부터는 다른 세션과 똑같이 답변할 수 있습니다. claude 세션 전용이며 같은 Wi‑Fi(LAN)의 승인된 기기에서만 가능하고, Codex 세션은 지원하지 않습니다.🔌 Connect session — sessions that can't take direct input (e.g. running in Windows Terminal, or already exited) show a “Connect session” button instead of the input bar. Tapping it reopens that session on the PC via claude --resume, after which you can answer it just like any other session. This works only for claude sessions, only from a device approved on the same Wi‑Fi (LAN), and does not support Codex sessions.
  5. 🔐 권한 승인 — Bash·파일 편집 등 위험 작업 요청 시 🔐 카드에서 허용/거부. 그 세션에 바로 적용됩니다.🔐 Approve permissions — for risky tools (Bash, file edits…) a 🔐 card lets you Allow/Deny, applied to that session.
  6. 🟡 응답 대기중 — 당신이 답해야 하는 세션은 목록에서 앰버색 카드로 표시됩니다. 탭하면 답변 화면이 다시 열립니다.🟡 Waiting for you — sessions needing your reply turn amber in the list; tap to reopen the answer sheet.
⏱️ 질문 답변 창이 지나도 계속 답할 수 있어요You can still answer after the reply window passes
Claude가 질문·권한을 기다리는 기본 약 10분(600초) 안에서는 답변이 그 세션에 즉시 반영됩니다. 이 창이 지나도 대기중 카드를 열면 같은 질문·답변 카드가 그대로 뜨고(Codex 세션 제외), 답변 보내기를 누르면 PC 세션 콘솔에 자동으로 입력됩니다 — 터미널을 직접 열 필요는 없습니다. 콘솔에 입력할 수 없는 상태면 실패 안내가 표시됩니다. 권한 승인은 창이 지나면 PC에서 처리해야 합니다. Within the ~10 minutes (600s) Claude waits for questions/permissions, your answer is applied to the session immediately. After that window, opening the waiting card still shows the same Q&A card (except for Codex sessions) — tapping Send answer types it straight into the session's console on the PC, no need to open a terminal yourself. If it can't reach the console, a failure notice is shown. Permission approvals still need to be handled on the PC once their window passes.
⚠️ Clawd on Desk와 동시 실행 주의Running alongside Clawd on Desk
두 앱이 같은 질문 훅을 함께 가로채면 답변이 전달되지 않습니다. Agent Hub 실행 시 Clawd on Desk가 켜져 있으면 종료할지 묻는 창이 뜹니다 — [예]면 닫고 계속, [아니오]면 유지하되 모바일에서 답할 때 “Clawd on Desk가 실행 중이라 답변을 전달할 수 없습니다” 안내가 나옵니다. If both apps intercept the same question hook, answers won't be delivered. On startup Agent Hub asks whether to close Clawd on Desk — [Yes] closes it; [No] keeps it, but answering from mobile then shows “Clawd on Desk is running, can't deliver the answer.”

🆚 차별점HOW IT DIFFERS

Claude Code 공식 원격 제어와 무엇이 다른가요?How is this different from Claude Code's official Remote Control?

Claude Code에는 세션을 원격에서 이어받는 공식 Remote Control 기능이 있습니다. 목표는 비슷하지만, Agent Hub는 연결 방식과 전제가 근본적으로 다릅니다. Claude Code has an official Remote Control feature for taking over a session remotely. The goal is similar, but Agent Hub differs fundamentally in how it connects and what it assumes.

📌 제한 사항 · 알아두기LIMITS & NOTES

쓰기 전에 알아두면 좋아요Good to know before you start

📶 알림엔 인터넷이 필요해요Alerts need internet
앱을 닫아도 오는 푸시는 폰·PC 모두 인터넷이 필요합니다(브라우저 푸시 서비스 경유). 질문 상세는 종단간 암호화되어 알림에 바로 표시됩니다. iOS는 홈 화면에 추가한 PWA + iOS 16.4 이상에서만 동작합니다. Push while the app is closed needs internet on both phone and PC (via the browser's push service). The question detail is end‑to‑end encrypted and shown right in the notification. On iOS it works only for a Home‑Screen PWA on iOS 16.4+.
✈️ 오프라인·다른 네트워크에서도 바로 실행Runs offline / on another network
PC가 꺼져 있거나 다른 네트워크일 때 앱을 열면, 아이콘 화면에서 멈추지 않고 마지막으로 봤던 세션 카드 목록으로 바로 들어갑니다(기기에 캐시된 상태). 서버에 다시 연결되면 자동으로 최신 내역으로 갱신됩니다. 이 기능을 쓰려면 같은 네트워크에서 한 번 이상 정상 접속해 화면이 저장돼 있어야 합니다. If the PC is off or you're on another network, opening the app no longer stalls on the icon screen — it goes straight to the last session‑card list you saw (cached on the device). Once it reconnects to the server it refreshes to the latest automatically. This needs at least one successful connect on the same network first so the screen is cached.
🌍 다른 네트워크에서 접속 (VPN)Connect from another network (VPN)
Agent Hub는 클라우드 서버가 없어 기본은 같은 네트워크 전용이지만, NetBird·Tailscale 같은 VPN을 쓰면 어디서나 접속할 수 있습니다. 설치·설정 단계, 모바일 접속, IP가 바뀌는 문제까지 위쪽 ‘VPN으로 어디서나 접속’ 섹션에 정리해 두었습니다. Agent Hub has no cloud server, so it's same‑network only by default — but a VPN like NetBird or Tailscale lets you connect from anywhere. Full setup, mobile access, and the changing‑IP question are covered in the ‘Connect from anywhere with a VPN’ section above.
🚀 Windows 시작 시 자동 실행 (기본 켜짐)Launch on Windows startup (on by default)
PC에 로그인하면 Agent Hub가 자동으로 실행됩니다(트레이에 상주). PC 콘솔 설정 › 일반의 ‘Windows 시작 시 자동 실행’ 체크로 언제든 끄고 켤 수 있어요. 관리자 권한은 필요 없으며(현재 사용자 로그인 항목에만 등록), 설치본에서만 적용됩니다. Agent Hub launches automatically when you log in to the PC (living in the tray). Toggle it anytime with ‘Launch on Windows startup’ under the PC console's Settings › General. No admin rights needed (registered only for the current user), and it applies to the installed build only.
🔄 재설치·업데이트해도 인증서·승인은 유지Reinstall/update keeps cert & approvals
인증서와 기기 승인은 LocalAppData에 보관되어 재설치·업데이트에도 유지됩니다(재승인 불필요). 홈 화면에 설치한 앱에서는 이미 인증서가 깔려 있으므로 헤더의 인증서 다운로드 버튼이 숨겨집니다. 완전 삭제 후 재설치하거나 인증서를 수동 삭제해 연결이 끊긴 경우, 연결 화면에 뜨는 ‘새 인증서 설치’ 링크(또는 앱에 표시된 주소)를 휴대폰 브라우저로 열어 다시 받으세요. 이 받기는 같은 LAN 안에서만 동작하며 HTTP로 제공되므로 신뢰하는 로컬 네트워크에서만 쓰세요. The certificate and device approvals live in LocalAppData, so they survive reinstalls and updates (no re‑approval). In the installed home‑screen app the cert is already trusted, so the header download button is hidden. Only after a full uninstall + reinstall or if you manually deleted the cert and lost the connection, re‑download via the “Install new certificate” link shown on the connection screen (or the address shown in the app) in your phone browser. That download works only on the same LAN and over HTTP, so use it only on a trusted local network.
🧱 첫 연결이 안 되면 방화벽First connection fails? Firewall
휴대폰에서 첫 접속이 안 되면 PC의 Windows 방화벽에서 해당 포트의 수신(인바운드) 허용이 필요할 수 있습니다. If the first connection from a phone fails, you may need to allow inbound connections for that port in the PC's Windows Firewall.
🟡 ‘응답 대기중’은 언제 뜨나요When does “Waiting for you” show?
사용자가 실제로 답해야 할 때(질문·권한·입력 대기)만 카드가 앰버색으로 표시되고, 서브에이전트 대기 등 배경 작업엔 표시되지 않습니다. 진행 중 권한요청 표시는 앱을 새로고침하면 초기화될 수 있습니다. The card turns amber only when you must respond (a question, a permission, or input); it is not shown for background work like waiting on a subagent. A pending permission highlight may reset if you reload the app.
🌗 라이트/다크 테마 전환Light / dark theme
헤더 맨 오른쪽의 🌙 / ☀️ 버튼으로 라이트·다크 테마를 전환할 수 있습니다. 선택한 테마는 기기에 저장되어 다음에 열 때도 유지됩니다. Use the 🌙 / ☀️ button at the far right of the header to switch between light and dark themes. Your choice is saved on the device and kept the next time you open the app.
📲 설치 안내 배너Install banner
앱을 설치하지 않고 브라우저로 접속하면 세션 목록 위에 설치 안내 배너가 뜹니다. 인증서가 아직 없으면 인증서 설치부터 안내하고, 준비되면 앱 설치 버튼이 표시됩니다. 홈 화면에 설치하면 배너는 사라집니다. If you open it in a browser without installing, an install banner appears above the session list — it points you to Install certificate first if needed, then shows an Install app button when ready. Once installed to your home screen, the banner disappears.
